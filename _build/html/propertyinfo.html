PropertyInfo¶

This page seeks to explain the methods and classes involved in the PropertyInfo sub-application of the Propertea application.

Important

Please note that the uses of these classes and functions are heavily dependent on the provision of the CSV files that store the transport data and other related information.

controller.py Methods¶

class propertyinfo.controller.PropertyInfoController(request)¶

This controller class executes the logic for the ‘PropertyInfo’ sub-application.

Variables

  • name – Name of property considered.

  • keyword – Search keyword used to find the building.

  • URL – URL for API.

  • info – Information Retrieved from API.

  • address – Address of Property.

  • X – X-Coordinate of Property.

  • Y – Y-Coordinate of Property.

  • LAT – Latitude of Property.

  • LONG – Longitude of Property.

  • MRT_LRT_Data – Data of the nearest MRT/LRT.

  • Bus_Data – Data of the nearest Bus Stop.

  • images – Data from Google Images.

  • request – The HTTP request at that instance.

favourite()¶

This function implements logic for toggling the view favourites button.

Returns

The list of Favourite Properties in the database for a particular user.

favouriteResponse()¶

This function implements logic for toggling the favourites button.

Returns

Redirect back to the search page.

getResponse()¶

Returns the ‘index’ page for the propertyinfo sub-application.

Returns

Render of ‘index’ page.

models.py Methods¶

class propertyinfo.models.BusData(X, Y)¶

This class aims to facilitate storing and retrieving Bus Stop data, while being an extension of Transport Data.

table()¶

This function aims to create the tables for the first and last bus services arriving at the selected bus stop.

Returns

A plot of the table of the first and last bus services arriving at the selected bus stop.

class propertyinfo.models.MRTLRTData(X, Y)¶

This class aims to facilitate storing and retrieving MRT and LRT station data, while being an extention of Transport Data.

table()¶

This function aims to create the tables for the first and last train services from the selected MRT and LRT station.

Returns

A plot of a table containing the first and last train timings from the selected MRT and LRT station.

class propertyinfo.models.PropertyImages(name)¶

This class aims to facilitate storing and retrieving property images.

Variables

name – Google Image Search Keyword

getLinks()¶

This function generates a link to view more Google Images on the selected property.

Returns

The link that can be used to link to a Google Images search.

getURL()¶

Returns the URL.

Returns

URL.

class propertyinfo.models.TransportData(X, Y, directory, volumeDir, routeDir, type, term)¶

This class aims to facilitate storing and retrieving transport data.

Variables

  • X – X-Coordinate of Property.

  • Y – Y-Coordinate of Property.

  • directory – Location of Stations CSV file.

  • volumeDir – Location of the Volume CSV file.

  • routeDir – Location of the Route CSV file.

  • type – “Bus” or “MRT”.

  • term – “Stop” or “Station”.

  • name – Transport Facility Name.

  • number – Bus Stop Number or MRT Number.

  • lat – Latitude of Facility.

  • long – Longitude of Facility.

plot()¶

This function seeks to plot the transport ameneties graph.

Returns

A plot of the traffic volume of the selected transport facility.

Note

Please note that we have separated view logic into controller and views in order to comply with the MVC framework.
