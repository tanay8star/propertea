Search¶

This page seeks to explain the methods and classes involved in the Search sub-application of the Propertea application.

Important

Please note that we have separated view logic into controller and views in order to comply with the MVC framework.

controller.py Methods¶

class search.controller.SearchController(request)¶

This controller class executes the logic for the ‘Search’ sub-application

Variables

  • request – The HTTP request at that instance.

  • keyword – The search keyword used.

  • filterby – The filter variable used, if present.

  • district – The district used to search, if used.

favourite()¶

This function implements logic for toggling the view favourites button.

Returns

The list of Favourite Properties in the database for a particular user.

favouriteResponse()¶

This function implements logic for toggling the favourites button.

Returns

Redirect back to the search page.

filter(result)¶

This function implements logic for filtering the results obtained.

Parameters

result – The result obtained via Search.

Returns

The filtered result based on the filter selected.

getResponse()¶

This function implements logic for displaying the index page.

Returns

The render of the search results.

search()¶

This function implements logic for the ‘Search Results’ page.

Returns

Calls the relevant functions based on user request and return the filtered result.

searchByDistrict()¶

This function implements logic for searching by district.

Returns

List of Properties in the district based on user district selection obtained from the CSV.

searchByKeyword()¶

This function implements logic for searching by keyword.

Returns

List of Properties related to the specific keyword.
